Web frameworks

The future of tools development

Purpose

Evaluate the leading frontend web frameworks for our next generation of tools.

Goals

  2. Evaluate a few promising frameworks
  3. Pick a framework for new tools

Frameworks

  • React*

Results

Angular: the good

  • Dev productivity: ng-cli
  • Reusability: components & services
  • Testability: karma + jasmine, protractor

Angular: the bad

  • Learning curve: notoriously complex and hard to learn, very large API surface, RxJS, Typescript, Zone.js
  • Documentation: bad at launch, has improved
  • Maintainability: many separate files
  • Community: fragmented, some moving from AngularJS, others transitioning despite Google backing

React: the good

  • Community/maturity: very large, stable, growing community backed by Facebook
  • Documentation: great official docs, lots of videos, courses
  • Dev Productivity: create-react-app, great devtools & IDE support, StatsApp
  • Testability: Jest
  • Maintainability: Separation of concerns, pure functional components

React: the bad

  • Learning curve: JSX, Redux, ES2015+ (better than Angular)

Vue: The good

  • Learning curve: known for minimalism, simplicity, easy to learn and use, familiar when coming from AngularJS (e.g. templates, v-* directives)
  • Developer productivity: vue-cli, great devtools, good IDE support, AngularJS familiarity
  • Maintainability: single file components
  • Documentation: great official docs

Vue: the bad

  • Community/maturity: newer than React (2014), much smaller but rapidly growing community, not backed by a large organization, could fade out
  • Testability: no dedicated tools, can use any JS testing library (e.g. karma + jasmine, mocha + chai)

Conclusions

  • Angular is out
  • React wins for community & maturity, safer bet
  • Vue wins for learning curve and dev experience, riskier option
